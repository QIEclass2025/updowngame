포켓몬 도감 업다운 게임(updown game) 프로젝트 보고서

본 문서는 Python Tkinter를 활용하여 개발된 '포켓몬 도감 업다운 게임'의 구조, 기능 및 실행 방법에 대해 기술합니다. 본 프로젝트는 고전적인 업다운(Up & Down) 게임 로직을 기반으로 하며, 외부 API를 연동하여 실시간 데이터를 시각화하는 인터랙티브 GUI 애플리케이션입니다.

1. 프로젝트 개요

프로젝트명: 포켓몬 도감 UP & DOWN 게임 (Survival Mode)

개발 언어: Python 3.12 이상

주요 라이브러리: Tkinter (GUI), Pillow (이미지 처리), urllib (HTTP 요청)

목표: 제한된 기회 내에 난수로 설정된 타겟 번호를 논리적 추론과 아이템 활용을 통해 맞추는 게임 구현.

2. 주요 기능 및 시스템 구성

2.1. 사용자 인터페이스 (GUI)

포켓몬 도감 디자인: 붉은색 본체와 듀얼 스크린 레이아웃을 채택하여 원작 게임기(Nintendo DS) 및 도감의 시각적 요소를 재현하였습니다.

반응형 피드백: 사용자의 입력에 따라 시각적 텍스트와 이미지, 상태 표시등이 실시간으로 갱신됩니다.

2.2. 게임 로직

범위 설정: 1세대(관동)부터 9세대(팔데아)까지의 포켓몬 도감 번호 범위를 프리셋으로 제공하며, 사용자 지정 범위 설정도 지원합니다.

업다운(Up & Down) 알고리즘: 사용자가 입력한 숫자가 정답보다 높은지 낮은지를 판별하여 탐색 범위를 갱신합니다.

자원 관리 시스템: '몬스터볼'로 명명된 제한된 기회(Life) 시스템을 도입하였으며, 오답 시 또는 아이템 사용 시 기회가 차감됩니다.

2.3. 전략 아이템 (Inventory)

게임의 전략적 요소를 강화하기 위해 3종의 아이템을 구현하였으며, 게임당 1회만 선택하여 사용할 수 있습니다. 단, 아이템 사용 시에도 턴(몬스터볼)이 1회 소모됩니다.

스코프렌즈: 현재 탐색 범위를 정답 기준으로 절반으로 축소합니다.

플러스파워: 3턴 동안 탐색 범위 압축 효율을 5% 추가 증가시킵니다.

자뭉열매: 소모된 기회(몬스터볼)를 3개 회복합니다. (최대치 초과 불가)

2.4. 외부 API 연동

비동기 데이터 처리: threading 모듈을 사용하여 메인 UI의 프리징(멈춤) 현상 없이 백그라운드에서 데이터를 호출합니다.

실시간 데이터 바인딩: 입력된 번호에 해당하는 포켓몬의 이름, 속성, 신체 정보, 도감 설명, 스프라이트 이미지를 실시간으로 가져와 출력합니다.

2.5. 데이터 영구 저장 및 관리 (Data Persistence)

단발성 게임 플레이에 그치지 않고, 사용자의 성취를 기록하기 위해 데이터 영구 저장 시스템을 구축하였습니다.

JSON 로그 시스템: 게임 승리/패배/도망 시 `pokedex_adventure_log.json` 파일에 플레이 기록(날짜, 포켓몬 정보, 시도 횟수, 사용 아이템, 결과)이 자동으로 저장됩니다.
기록 열람 및 초기화: 메인 화면과 게임 내 메뉴에서 언제든 '모험 기록(PC 박스)'을 열람할 수 있으며, `Treeview` 위젯을 통해 표 형태로 시각화됩니다. 또한, 사용자 편의를 위해 기록 초기화 기능을 제공하며, 실수로 인한 삭제를 방지하기 위해 재확인(Confirm) 팝업을 구현하였습니다.

2.6. 크로스 플랫폼 호환성 (Cross-Platform Compatibility)

다양한 운영체제 환경에서의 실행 안정성을 보장하기 위해 시스템 환경 감지 로직을 적용하였습니다.
OS 감지 및 폰트 매핑: `platform` 모듈을 활용하여 실행 환경이 Windows인지 macOS(Darwin)인지 판별합니다.
동적 폰트 할당: Windows 환경에서는 `Malgun Gothic`, macOS 환경에서는 `AppleGothic`을 자동으로 할당하여, 운영체제 차이로 인한 폰트 깨짐이나 UI 틀어짐 현상을 방지하였습니다.

2.7. 메인 화면 동적 연출

랜덤 스프라이트 로딩: 애플리케이션 실행 시마다 메인 화면에 무작위 포켓몬 이미지를 로드하여, 매번 새로운 시각적 경험을 제공하고 '도감 데이터 수신 중'이라는 컨셉을 강화하였습니다.

3. API 활용 명세

본 프로젝트는 오픈 소스 API인 PokeAPI를 사용합니다.

API Endpoint: https://pokeapi.co/api/v2/

인증 방식 (Authentication):

PokeAPI는 Public API로서 별도의 API Key나 OAuth 인증 토큰을 요구하지 않습니다.

따라서 본 프로젝트는 실행 시 별도의 환경 변수(.env) 설정이나 키 발급 절차가 필요하지 않습니다.

활용 데이터:

/pokemon/{id}: 포켓몬의 기본 정보(이름, 타입, 키, 몸무게) 및 스프라이트 이미지 URL

/pokemon-species/{id}: 다국어 지원 이름 및 도감 설명(Flavor Text)

/item/{id}: 아이템 스프라이트 이미지

4. 개발 환경 및 의존성

본 프로젝트는 패키지 매니저 uv를 통해 의존성을 관리합니다.

4.1. 필수 라이브러리

python >= 3.12

pillow: 이미지 리사이징 및 GUI 렌더링 처리를 위해 사용됩니다.

tkinter: Python 표준 GUI 라이브러리입니다.

4.2. 실행 환경 설정

프로젝트 실행을 위해서는 다음 절차를 따릅니다.

저장소 복제 (Clone)

git clone [레포지토리 주소]
cd [프로젝트 폴더]


의존성 설치 및 동기화
uv를 사용하여 pyproject.toml에 명시된 환경을 구성합니다.

uv sync


애플리케이션 실행

uv run updown_game.py
